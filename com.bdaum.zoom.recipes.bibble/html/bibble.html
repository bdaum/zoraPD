Bibble (AfterShot) Support

ZoRa Photo Director is able to reflect most of the development settings for RAW and DNG files made with the Bibble 5/AfterShot RAW converter. (Development settings of Bibble 5/AfterShot 5 plug-ins are not supported.)

These settings (recipes) are stored in an external XMP file located in the same folder as the original image (sidecar). ZoRa Photo Director is able to read these development recipes and to apply them when importing RAW files, refreshing or viewing them, or when deriving images from them during export operations.

To enable this support go the Window>Preferences>Import>Raw Conversion* and check the Bibble/AfterShot recipe detector. Make also sure that recipe detection is enabled (Process recipes when importing RAW images).

Because XMP sidecars are used by some other RAW converters, too, you may specify an XMP priority to resolve conflicts between competing recipes. The smaller the number, the higher is the priority.

You must configure Bibble/AfterShot to store its development data in XMP meta files. To do so set in Bibble/AfterShot the option "Allow Bibble/AfterShot .xmp creation..." .

Bibble/AfterShot saves this information incrementally in the moment a change is applied. When the image happens to be located in a watched folder (see Catalog Properties), ZoRa Photo Director will apply these changes either as soon as Bibble applies changes to another image (indicating the end of the editing session for the current image) or when control returns to ZoRa PD.
