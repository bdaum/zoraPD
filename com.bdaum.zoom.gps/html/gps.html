Working with GPS data

GPS data can be very useful for landscape photos and for images taken on locations. By "tagging" images with GPS data it is later easy to find out where an image was taken. 

GPS coordinates can be stored in the EXIF data of an image. 

The minimum GPS data consist of a pair of numbers, one for the geographic latitude, the second for the longitude. It is possible, to specify these data manually but the most convenient way is to import the data generated by a GPS Tracker - in ZoRa Photo Director just drop the GPX or NMEA file onto an image collection.

Alternatively, you can manually geo-tag images in the Map view. Optionally, geo-naming (reverse geo-coding) is performed automatically with the help of web service (geonames.org or Google Places).

Locations

The IPTC data of an image can store several locations: the camera locations and the locations shown. While the camera location if often supplied by using a GPS tracker, the locations shown must be entered manually (IPTC>Origin>Location created). ZoRa Photo Director maintains a list of already defined locations for quick selection (Catalog>Properties>Locations).

Expoiting geographical information

Once an image is geotagged, you can utilize this information in various ways:
  • Display image locations in the Map view.
  • Search for location names.
  • Search for images in the proximity of a given image.
  • Specialized web services such as flickr, locr, SmugMug, or Google Earth can exploit GPS information and place uploaded images onto the right map position.
