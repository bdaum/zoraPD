Geo-tagging

Some cameras can geotag images when taking pictures. If your camera does not do this, you can still geotag images by using data from a GPS tracker or a GPS navigation device. This data is matched against the timestampin the images EXIF data.
This process can be invoked via the menu function Metadata>Geotagging or simply by dropping a supported geo-tracking file onto a collection or into a gallery area. Supported are GPX files (.gpx) and NMEA files (.nmea or .log).
Alternatively, you may geo-tag manually by setting geo-markers in the Map view.

Before doing so, you should configure the Geo-tagging facility by setting options through Window>Preferences>GPS:
General
Timing
Timeshift
You may specify a time shift value if the clocks of camera and tracker were not set to the same time. The time shift value will be added to the camera time before matching the image to the tracker recording. The best practice is setting the timezones of the camera to the timezone of your home computer. The clock of the tracker is set by the satellite signal (UTC = Universal Time Coordinated). So, if your camera is set up correctly, timeshift should reflect the difference between UTC and current local time. Press the "Restore to defaults" button to find out the current value (value may change due to daylight savings time).
Tolerance and Interval
A tolerance value is useful to match images to tracker entries that do not match exactly, for example, if you switched the tracker on after taking pictures. The position of an image will be interpolated between the trackpoints whose timestamps are closest to the timestamp of the image.
Mode
Overwrite existing geotags
Check this option if you want to refresh the geo-tags of images that have already been geo-tagged.

Use Waypoint information
Check this option if you want to utilize GPX waypoint information. Trackpoints close to a waypoint will be resolved using the waypoint information instead of using web services when geo-naming.
Update altitude through webservice
This will usually result in more precise altitude readings but will slow down the process of geo-tagging.

Edit trackpoints
Mark this option if you want to edit the trackpoints contained in imported trackfiles before applying these trackpoints to your images. 
Keywords
Include coordinates into keywords
Check this option if you want to include the GPS coordinates into the image IPTC keywords. This can be useful if you want to allow keyword search on coordinates (especially with third party applications). Without this option the GPS coordinates are only stored into the EXIF meta data of the image. By default, this option is switched off in order not to overcrowd the keywords entry.

Include place names into keywords
Check this option if you want to include the geonames into the image keywords. This can be useful if you want to allow keyword search on place names. By default, this option is switched on.
Webservices
Here you can configure the webservices used for geonaming and other purposes
Used Geonaming Webservice
Select a geonaming webservice from the displayed list. Geonaming determines place names from location coordinates.

Max. request per hour
The webservices used for geonaming and elevation (Update altitude through webservice) are all limited in capacity depending on the provider and on your agreement with the provider. By limiting the request per hour you can make sure not to run out of capacity during a geotagging and geonaming task. The operation is simply slowed down for not exceeding the hourly limit.
Please be aware that elevation request are always handled through geonames.org independent of you setting in Used Geonaming Webservice.
