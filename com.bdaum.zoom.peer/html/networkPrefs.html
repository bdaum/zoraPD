Networking Preferences

The networking preferences are invoked via Window>Preferences>Network*.

Own address

  • Here you find your own network address.
    • Computer name (IP address). For information only. This field cannot be modified.
    • Port. The communication port used. Normally you should use the provided default value. However, in some cases the port may already be in use, for example if another peer is running on the same computer. In this case choose a different value. You must restart the application after a port change.

Shared catalogs

Here you list all catalogs that can by accessed by other peers but also by yourself when performing a network search across multiple peers and/or multiple catalogs.

  • Use the Add button to add a new catalog to the list. That can be the currently opened catalog but also any other .zdb catalog on your computer. By clicking the new entry in the Privacy column you can adjust the privacy setting for the catalog:
    • All. All images in the catalog can be accessed by remote peers.
    • Medium. Only images classified as public or medium can be accessed by remote peers.
    • Only public elements. No restricted images can be accessed by remote peers.
    • Local. The catalog is not accessible by remote peers. This is the typical setting for performing local searches across multiple catalogs without opening to catalog to remote peers.
  • Use the Access Rights button to reduce or extend the access rights of specific remote peers to the selected catalog. (You cannot add restrictions to catalogs with the Privacy setting "Local").
    By default each peer can search a catalog and view the results in full resolution.
    • Mark Search to allow the peer search the catalog.
    • Mark View to allow the peer to view items in full resolution
    • Mark Voice note to allow the peer to replay voice notes
    • Mark Copy to allow the peer to copy an item to his own node. (If Voice note is not marked, attached voice notes are not copied.)
    Removing all marks will close the catalog for the specific peer.
    Note: Access rights definitions will always affect the complete peer computer but not a single port!
  • Use the Remove button to remove the selected access rights or the selected catalog from the list.

Peer nodes

  • Peers included in network search operations
    Here you set up a list of all peers that you want to include into a network search:
    • Use the Add button to add a new peer to the list. Specify the peer's computer name and communication port. If you want to use an IP address instead of the computer name make sure that the peer uses a fix IP address. Some operation systems allocate a different IP address after each new start.
    • Use the Remove button to remove the selected peer definition from the list.

Incoming calls

Here you can monitor which other peers have accessed your catalogs. If you do not want a certain peer to access your catalogs, simply select him in the list and click the Toggle blacklist button. Another click re-establishes the access rights of that peer.

Note: Blacklisting will always affect the complete peer computer but not a single port!

By clicking the Clear button you can remove all entries from the list - except the blacklisted entries.
