Networking concepts

Provided that the networking component had been installed, many search dialogs show a "Search in network" button.  When checking this option, searches are performed simultanously across multiple peer nodes and catalogs. Otherwise, only the currently opened catalog is searched.

Network searching is possible with

  • user defined permanent collections
  • adhoc collections and other adhoc queries such as time search, similarity search, etc.

Network searching is NOT possible with

  • system collections (time line, directories, etc.)
  • albums
  • person albums

Providers and Consumers

Each peer nodes acts both as a provider and as a consumer:
  • a provider provides the access to one or several of his own catalogs. Each provider can define different privacy levels for the shared catalogs and can restrict or extend the access rights for specific consumer/catalog combinations. See Window>Preferences>Network>Shared Catalogs*
  • a consumer can have access to catalogs offered by other providers. For this purpose he defines a list of peers that should be accessed during a network search. 
    See Window>Preferences>Network>Peer nodes*

Remote and local searching

Network searches are performed differently on local and remote nodes:
  • On remote peer nodes all shared catalogs not specified as "Local" or searched providing the particular consumer is allowed to perform a search. The result is filtered according to each catalogs privacy settings.
  • On the home node the currently opened catalog is searched as well as all shared catalogs without any restrictions.

Non-network searches are only performed on the currently opened catalog.
