LightZone Support

The free image editor and raw converter LightZone works a bit different from other raw converters. Recipes are stored in JPeG files, the filename suffixed with "_lzn". This JPeG image is just a thumbnail of the orignal image with the recipe applied. It also contains a pointer to the original image.

This has advantages and disadvantages:

  • The thumbnail saves us from reapplying the processing described by the recipe. By importing both the original and the _lzn.jpg file into ZoRa Photo Director you can see both the original state and the processed state of an image.
  • However, the LZN files don't like the original to be moved or renamed. The pointer in the LZN file becomes invalid and the LZN can no longer be edited with LightZone. In  ZoRa Photo Director this would affect the operation Move, Rename and Import folder into new Structure.  ZoRa Photo Director solves this problem. solves this problem by automatically updating the pointers in the LZN files when an original is moved or renamed. 
    Hint: When importing both original and LZN-files into a new folder structure make sure this happens within the same session.

ZoRa Photo Director is able to detect the relationships between original files and derived LZN files and to display those relationships in the Hierarchy view. To do so mark the respective option in Window>Preferences>Import>General*.
