Text Retrieval

Text retrieval (Image>Text search) is quite similar to a Google search and allows searching for text phrases in images regardless if the phrase occurs in the keywords, in a title, a description, a slide, exhibit, or web exhibit using that image, etc. Text retrieval allows for very quick text searches but does not have the precision of a specific query such as a keyword query. For example, if you search for "Peter" you will find all images with a keyword "Peter" but also all images where "Peter" is found in the author, owner, or copyright field - or any other field, that is.
However, by combining multiple sub queries into one query expression you can narrow down your search results:
  • Search for phrases by including them into double quotes, e.g. "uncle tom".
  • Use wildcard operators such as * and ? (not within phrases). * stands for any string, ? for a single arbitrary character.
    Note: You cannot use a * or ? symbol as the first character of a search.
    Examples: ani* would find all words starting with 'ani'. 
  • Postfix a word with ~ to perform a similarity search. This can be useful to get search results even in the case of spelling errors.
  • Use the AND operator for returning images containing both words or phrases, for example: people AND animals.
  • Use the OR operator for returning images containing either the left or the right word or phrase but not both: people OR animals
  • Use the NOT operator for excluding images containing a word or a phrase. For example people NOT animals would return images tagged with 'people' but not tagged with 'animals'. Alternatively, you can write people -animals.
  • Use parentheses for complex search expressions, for example: (people AND animals) OR (cars AND animals)
  • Some characters contained in a search word or phrase must be escaped: + - && || ! ( ) { } [ ] ^ " ~ * ? : \
    Prefix such characters with \, for example: wow\!
