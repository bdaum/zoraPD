Dialog Ähnlichkeitsrecherche

Dieser Dialog wird verwendet, um ein Beispielbild und Suchoptionen für die Ähnlichkeitssuche zu spezifizieren.
Content Based Image Retrieval (CBIR) ist ein seit langem etabliertes Forschungsgebiet. Es gibt inzwischen viele bekannte Suchmethoden (Algorithmen). Wir haben einige moderne Suchalgorithmen integriert, die besonders für fotografische Bilder geeignet sind. Sie können  eine geeignete Suchmethode in den Benutzereinstellungen auswählen.
Keiner dieser Algorithmen - bis auf die Clarifai-Algorithmen -  ist tatsächlich in der Lage, den Sinn eines Bildes zu erfassen. Zum Beispiel erkennt der Algorithmus nicht die Bäume in einem Waldbild, sondern wertet visuelle Eigenschaften wie Grundfarbe, räumliche Verteilung der Farbe, Farb-Histogramme, Texturen usw. aus. Sie sollten also von dieser Funktion keine Wunder erwarten  - sie ist kein Ersatz für die Schlagwortsuche. Aber in vielen Situationen ist sie dennoch nützlich, besonders, wenn sie mit der Schlagwortsuche kombiniert wird.
Die Clarifai-Algorithmen (allgemeine Ähnlichkeit und Gesichtsähnlichkeit) testen dagegen nicht auf visuelle Ähnlichkeit, sondern auf Ähnlichkeit von semantischen Bildkonzepten oder von Gesichtern, die in dem Bild gezeigt werden. Der Test auf Gesichtsähnlichkeit funktioniert sogar für Personengruppen.

Ähnlichkeitsrecherchen können lokal oder über ein Netzwerk ausgeführt werden. Sie können auch auf die aktuelle Kollektion eingeschränkt werden, so dass sie auf diese Weise die Ähnlichkeitsrecherche mit einer Metadatensuche (Kollektion) kombinieren können. Nach der Ausführung einer Ähnlichkeitsrecherche können Sie diese als Live-Kollektion abspeichern. Dazu rufen Sie im Galeriesichtfenster die lokale Menüfunktion Abfrage als Kollektion speichern auf.

Hinweis: Bei einer vernetzten Gruppe, in der Suchanfragen über das Netzwerk gestellt werden, sollten alle Katalog die selben Suchalgorithmen unterstützen.

Hinweis: Ein vereinfachte Suchmethode besteht darin, ein Bild auf das Suchfeld in der Menüleiste zu ziehen.

Referenzbild

Die Suche basiert auf einem Referenzbild, das im Dialog präsentiert wird. Wenn kein Bild ausgewählt wurde, wird ein weißer Hintergrund gezeigt. Sie können mit Hilfe der Malwerkzeuge (Bleistift, Airbrush, Pipette) eine Zeichnung erstellen oder aber das Beispielbild abändern.

Suchalgorithmus

Welcher Algorithmus der beste ist, hängt sehr stark vom Charakter Ihrer Bilder ab. In einem Katalog der überwiegend Texturen enthält, zum Beispiel, sollten Sie das Tamura-Textur-Filter verwenden. Bei durchschnittlichen fotografischen Inhalten sind die Methoden CEDD und FCTH (oder auch deren Kombination JCD) recht effizient. Das Auto-Farbkorrelogramm ist ein moderner Algorithmus, der ebenfalls sehr gute Ergebnisse bringt.
Beachten Sie, dass nur Algorithmen für die Suche verwendet werden können, die von Ihrem Katalog unterstützt werden. Welche das sind, können Sie in den Katalogeigenschaften konfigurieren. Wenn Sie hier viele Algorithmen auswählen, haben Sie zwar bei der Bildsuche viele Wahlmöglichkeiten, verlangsamen aber die Indizierung und belegen mehr Platz auf der Harddisk.

Jede Abfrage liefert zu jedem Bild eine Punktzahl zwischen 0 und 100 Prozent. Sie können die Menge der zurückgegebenen Bilder entweder nach der Anzahl begrenzen oder durch die Angabe einer Mindestpunktzahl, die von jedem Bild erfüllt sein muss.

Zusätzliche Schlagworte

Wenn Sie eine Schlagwortgewichtung von mehr als 0 spezifizieren, können Sie die visuelle Ähnlichkeitssuche mit einer Schlagwortsuche kombinieren. Dabei werden die Schlagworte dem aktuell ausgewählten Bild entnommen, oder können von Hand eingegeben werden. Je mehr Schlagworte eines Bildes mit den vorgegebenen Schlagworten übereinstimmen, umso höher ist die zurückgegebene Punktzahl. Um nur eine visuelle Suche durchzuführen, spezifizieren Sie eine Schlagwortgewichtung von 0. Um nur nach Schlagworten zu suchen, spezifizieren Sie eine Schlagwortgewichtung von 100.
