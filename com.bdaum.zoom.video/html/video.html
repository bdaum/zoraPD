Video concepts

When the Video Support component was installed, ZoRa PhotoDirector is able to include video files into the catalog, too. The handling of video files is quite similar to the handling of photos. However, some features are not supported for video files.

Video engine

ZoRa PhotoDirector uses the VLC video player (VideoLAN client) as an internal video engine. VLC is a very robust video engine accepting almost any video format.
VLC is not included in the ZoRa PhotoDirector distribution. In many cases you will already have a VLC installation on your computer. Otherwise you must download and install it.
Finally, you must tell ZoRa PhotoDirector where the VLC executable is found. (Window>Preferences>Import>Video).

Features not supported for video files

  • Exhibitions
  • Slide show
  • Web galleries
  • Export to PDF
  • Export to photo communities, but there are export function for video communities
  • Export to KML

Remote video files

Other than for images, remote video files are not supported.

Metadata

Unfortunately, there is no universal metadata standard for video file formats similar to the EXIF metadata for photos. Therefore we treat metadata for video in the following way:
  • Metadata which can be derived from file properties such as file size, URI, etc, and ZoRa specific metadata such as rating, privacy, color code, etc, are shown in the Properties group. ZoRa specific metadata can be exported into XMP-sidecars.
  • A few image related properties such as frame size are shown in the EXIF group.
  • Video specific metadata derived from metadata entries embedded in the video stream are shown in the Video group. These entries are strictly read-only.
  • Addidtional information such as keywords, categories, locations, etc can be entered in the IPTC group. This metadata can be exported into XMP-sidecars, too.
