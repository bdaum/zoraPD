Image Processing

SInce Version 3, ZoRa PhotoDirector makes use of the excellent twelvemonkeys image library. This library covers practically all of the popular image formats and many more. For some additional file formats we use the services of JAI (Java advanced imaging), Jimi (Java Image Management Interface), and ImageJ. For information about RAW processing look here.

Image size

There is no limit on the image size. However, since ZoRa PhotoDirector keeps images completely in memory, large image are subsampled. When this happens depends on the main memory available. In consequence you cannot enlarge this images to pixel level when using the internal image viewer. If you require this feature, please embed an external image viewer (Windows>Preferences>External Programs*)).

JPEG 2000

Unfortunately, the JPEG 2000 license is not compatible with the GNU public license under which ZoRa PhotoDirector is distributed. Therefore, by default, ZoRa PhotoDirector comes without JPEG 2000 support. However, if you require JPEG 2000 support, you can download com.bdaum.zoom.image.jpeg2000_1.2.0.jar from here, and place it into the folder ZoRa/plugins/ within your installation folder. Then restart the application.
Note, that you must repeat this process after re-installing the application.

WebP

WebP is a relatively new, Google-developed, open-source-based and free format for image files. At the same quality, it compresses images much better than JPeG. WebP is currently used mainly in the Goggle world (Chrome, Android), but also by some other providers (Facebook, NetFlix, Opera). ZoRa PhotoDirector can import and export WebP files (export to folder or HTML).
