Files and Volumes

Images and videos are identified by their file path. Since this is not unique on all platforms, the volume name is also used.

For example, on Windows the file path consists from the drive letter, followed by a series of folders and subfolders. The problem here is external disks, which usually are assigned the same drive letter by Windows. Since ZoRa PhotoDirector can also catalog images from external data media, the name of the volume (CD, DVD, USB stick, SD card) is also used for identifying images.
It is therefore important to always provide such data carriers with a unique name during formatting!

On Linux and Mac, there are other problems. Here, the volume name is part of the file path, but the syntax of the file path is different from platform to platform. Thus, file formats of external media on Ubuntu have the form

/media/{volume}/...

On the Suse system

/run/media/{user}/{volume}/...

On MacOS

/Volumes/{volume}/...


The corresponding path patterns are defined in the file volumePatterns.ini in the plugin com.bdaum.zoom.image and can be changed as required. Regular expressions are used to define these patterns.

The default definition of this file is:

# MAC
/Volumes/(.*?)(?:/.*)?
# UBUNTU
/media/(.*?)(?:/.*)?
# SUSE
/run/media/.*?/(.*?)(?:/.*)?

It should be noted that these patterns are also used when migrating files to Linux or Mac systems (Catalog>Maintenance>Migrate catalog ...).
