Working with RAW files

The RAW image format is popular with many serious photographers. The reason for this is that unlike the limited and lossy JPEG format, RAW files contain all data captured by the camera sensor.

RAW images can not be directly displayed or printed. Instead, they need to be developed into a traditional image format such as TIFF or JPeG using a raw converter, such as Adobe Camera Raw, Adobe Lightroom, Capture One, RawTherapee, AfterShot, LightZone, and others.

Unfortunately, almost every camera manufacturer uses their own RAW format. In an attempt to standardize the RAW image format, Adobe has created the DNG format (Digital Negative). Some camera manufacturers support this format directly, but in most cases the images supplied by the camera must be converted to DNG files using Adobe DNG Converter. These DNG files must also be "developed" with a RAW converter to display images. However, not all RAW converters can convert DNG files obtained with the Adobe DNG Converter. Therefore, we recommend saving both the original RAW images and the derived DNG files.

Basic RAW support

ZoRa Photo Director can directly import RAW and DNG images. Optionally, you can have RAW images converted to DNG images, as ZoRa Photo Director can control the Adobe DNG Converter. (Note that the Adobe DNG Converter is not included with ZoRa Photo Director, you must download it from Adobe and install it separately.)

By default, ZoRa Photo Director internally uses the popular DCRAW module to convert RAW images into a displayable format (see also Importing Images). This is required each time a thumbnail is generated from an image when the images are viewed or exported. Normally, the conversion takes a few seconds. As new digital cameras appear on the market, DCRAW frequently changes its versions - more often than ZoRa Photo Director. However, you can simply put new DCRAW versions in the dropins/ subfolder in the ZoRa Photo Director installation folder to ensure import of new camera models.

Unfortunately, DCRAW has been discontinued. An alternative is the program dcraw_emu from the LIBRAW project. You can use this program with ZoRa Photo Director, too, but make sure that under Windows the dynamic library libraw.dll resides in the same folder as dcraw_emu.exe.

ZoRa Photo Director can also use RawTherapee or darktable in batch mode as raw converters. The results are better in quality when compared to DCRAW, and if you use RawTherapee or darktable primarily as a raw converter, it makes sense to integrate it into ZoRa Photo Director to keep the conversion results consistent. In addition, newer versions of RawTherapee or darktable are readily available on OS X platforms, too. RawTherapee even offers the possibility to support new camera types relatively easily - see here!

Here you can set up the RAW and DNG conversion: Window>Preferences>Import>RAW Conversion*.

ZoRa Photo Director can automatically track the relationships between RAW images and derived images. You can visualize these relationships with the Hierarchy view.

Raw Recipe Refection (3R)

When developing RAW images, the RAW converter does not change the original image. Instead, certain transformations are applied to the original image and the result stored in a conventional image format such as TIFF or JPEG. These transformations (recipes) are usually stored in separate files, such as XMP sidecars or in separate folders, so that later these transformations can easily be reapplied or modified.

ZoRa Photo Director is able to read these recipes and apply them to the image when the image is converted to a visible format. This is optional and can be configured under Window>Preferences>Import RAW Conversion*. In case there are multiple recipes from different RAW converters for an image, ZoRa Photo Director will use the latest recipe. In the rare case that two RAW converters use the same recipe files (as in the case of Adobe Camera Raw / Adobe Lightroom and AfterShot), the priority set in Window>Preferences>Import>RAW Conversion* decides.

RAW recipes that were used to create derived images (such as TIFF or JPEG images) can be automatically archived in the catalog (Window>Preferences>Import>RAW Conversion*). So you can later restore the original recipes as needed. To do this, call up the Hierarchy view for the RAW image (Relationships>Display Derivatives) and double-click the derived image in it. In the following dialog, click on Restore.

However, the above applies only if you use DCRAW as a raw converter. When using RawTherapee, you only have the option of using .pp3 sidecar recipes or the default processing profile set in RawTherapee. Recipes of other raw converters are ignored. The advantage though is that the recipe processing is done directly by RawTherapee so that the conversion results are 100% consistent with those of RawTherapee.

Advanced features: CHDK support

The CHDK is a tool that loads onto a memory card and can be used on most Canon consumer cameras. It allows you - among other things - to deliver images as RAW or DNG files. Unfortunately, this RAW format does not include the EXIF ​​data, and the DNG files contain only an incomplete set of EXIF ​​data. Luckily, the camera also produces an associated JPEG file with a complete set of EXIF ​​data.

To import the complete set of metadata with these files, we recommend the following procedure:
  • Set up the camera to deliver DNG images (CHDK menu). This allows you to process the images with most RAW converters (the RAW converter support for the proprietary CHDK RAW format is very limited).
  • Set up the camera so that image names are prefixed with IMG_ (CHDK menu).
  • When importing the images from the memory card with ZoRa Photo Director, enable the option Prefix for EXIF ​​Transfer under Advanced Options (see also Import from Device dialog). ZoRa Photo Director then imports the image from the DNG file and the EXIF ​​data from the associated JPEG file.
