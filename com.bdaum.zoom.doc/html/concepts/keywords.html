Keywords, Vocabularies, and Categories

Keywords and Categories are essential for classifying images. Both are supported by IPTC metadata. Now, what is the difference between keywords and categories?
  • Classification by categories is more general while the classification by keywords is more specific. For example, the category "Nature" or the subcategory "Mushrooms" covers a rather large field while the keyword "Boletus aestivalis" pinpoints to a precise species.
  • Categories are organized in hierarchies, for example
        Nature
            Mushrooms
            Plants
                Trees
            Animals
    In ZoRa Photo Director all categories will show up as collections in the Catalog view. Because of the hierarchical structure the collection "Plants" would also contain images from the collection "Trees".
    For keywords no such hierarchy exists. By default, no collections are generated for the many keywords that may exist but you may define your own collections for selected keywords.
  • Typically an image can belong to exactly one main category. The IPTC standard allows in addition Supplemental Categories.
    At the other hand, each image can be attributed with an unlimited number of keywords.
  • When a new catalog is created, a hierarchy of categories is automatically set up, depending on the catalog theme (see Catalog Properties) and based on the definitions in internal categories*.ini files. You may, however, overwrite these definitions by placing such files into the directory dropins within the installation folder:
        categories.ini
        categories_food.ini
        categories_travel.ini
        categories_wedding.ini
    You may also later modify the category hierarchy interactively in the Catalog Properties.
    A convenient way to create such files is to save the current category setup into a text file (Catalog Properties) and then modify it with a text editor.

    In addition, files keywords*.ini may be added to folder dropins for populating a new catalog with keywords. For the file format, see below.
A big problem with keywords is to avoid keyword cluttering because this makes it difficult to retrieve images by keywords. One remedy is to spell check all keywords which is done automatically in ZoRa Photo Director if a dictionary is defined  (see Catalog Properties).
Another remedy is to use Controlled Vocabularies. These can be defined in the Catalog Properties. If controlled vocabularies are defined, ZoRa Photo Director will flag all keywords that are not contained in a controlled vocabulary. On the Internet there a many predefined controlled vocabularies, for example here
    http://lightroom-keyword-list-project.blogspot.de/p/get-list.html
or here
    http://www.photo-keywords.com/specialist-list-download.php
You may of course roll your own, too.
The file format for categories, keywords, and vocabularies is almost identical and compatible with other application such as Adobe Lightroom.

  • Each category or keyword is contained in a separate line.
  • Hierarchies are defined by indenting a subitem with a TAB character.
  • Synonyms (used only for categories and vocabularies) are declared by enclosing an item with curly brackets.
  • Groups (used only in vocabularies) are declared by enclosing an item with brackets.
