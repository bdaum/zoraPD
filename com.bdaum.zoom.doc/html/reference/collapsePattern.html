Collapse pattern dialog

The Collapse Pattern dialog is invoked by pressing CTRL and clicking on the Collapse toolbar button in the Lightbox view.

The Collapse Pattern dialog defines how images with similar file names are collapsed to a single thumbnail. This is done by defining a typical file name pattern using wildcards such as * and ?.
One or several wildcards can be prefixed with a slash (/) for identifying relevant file name sections. Images are grouped together by comparing these relevant parts of a sequence of images.
It is possible to specify alternative patterns by concatenating them with a semicolon (;).

Typical examples:

/*.*Images are grouped by their plain file name (without file name extension)
e.g.  img0001.jpg is grouped together with img0001.tif.
/*_v*.*Images are grouped by their name (without version number and without file name extension)
e.g.  img0001_v01.jpg is grouped together with img0001_v02.tif.
/*_v*./*Images are grouped by their name and name extension (without version number)
e.g.  img0001_v01.jpg is grouped together with img0001_v02.jpg but not with  img0001_v01.tif.
/*_v*.*;/*.*A combination of the first two examples. img0001_v01.jpg is grouped together with img0001.tif and  img0001_v02.jpg.

In addition images belonging to a named stack can be collapsed. The option Collapse named stacks, too is equivalent to the pattern STACK#/*!*
