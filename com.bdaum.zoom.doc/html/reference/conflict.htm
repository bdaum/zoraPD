Import Conflict Resolution dialog

When importing images it may happen that certain images were already imported previously. This conflict can be resolved in several ways:

  • Skip image. This option will skip the image to be imported and move on to the next image to import.
  • Overwrite catalog entry.  The conflicting catalog entry is completely deleted before the image is imported again.
  • Synchronize with catalog entry. The catalog entry is overwritten only in parts by the new import. Which parts are overwritten can be selected with the following options:
    • Refresh Thumbnail. This option will update the catalog thumbnail and is typically used after images have been updated with an image editor or after the thumbnail size or sharpening had been changed. However, refreshing will be much faster if this option is switched off.
    • Refresh image status. De-select this group if you have manually modified status, color code, rating, content type, privacy settings, or user fields and don't want to reset that information.
    • Refresh EXIF data. De-select this group if the EXIF data have not changed. For example, you may only want to refresh the GPS data.
    • Refresh GPS data. De-select this group if you have geo-tagged the image and you do not want to reset that information.
    • Refresh IPTC data. De-select this group if you have manually entered keywords, categories and other IPTC data and you do not want to reset that information.

You may resolve these conflicts step by step, or apply the chosen policy to all future conflicts within the current import process by checking the option Apply to all conflicts. In addition, you may mark Only newer items, too, to process only items that have been modified after being previously imported into the catalog.
