Web Gallery Generator

Identifier:
com.bdaum.zoom.ui.galleryGenerator

Since:
0.5.4

Description:
Extension point to add different web gallery generators.

Configuration Markup:

<!ELEMENT extension (generator)>

<!ATTLIST extension

point CDATA #REQUIRED

id CDATA #IMPLIED

name CDATA #IMPLIED

>

Extension for adding generators for web galleries. At maximum one generator per plug-in.



<!ELEMENT generator (parameter*)>

<!ATTLIST generator

id IDREF #REQUIRED

name CDATA #REQUIRED

description CDATA #IMPLIED

class CDATA #REQUIRED

slideshows (true | false)

sections (true | false)

unsupported CDATA #IMPLIED

maxImages CDATA #IMPLIED

>

The generator responsible for generating a web gallery


  • id - globally unique generator id
  • name - display label of the generator
  • description - A textual (HTML) description of the generator. The available area is 300 x 500 pixels.
  • class - The class implementing the generator. Must extend com.bdaum.zoom.operations.internal.gen.AbstractGalleryGenerator
  • slideshows - Specifies if the generated gallery will be able to run slideshows
  • sections - Specifies if the generated gallery can be organized into sections (storyboards)
  • unsupported - A list of unsupported features separated by blanks. The following feature names can be listed: download showmetadata thumbnailsize opacity padding sectionfont captionfont navfont
  • maxImages - The maximum number (integer value) of images within a gallery. If this parameter is omitted, the maximum number is unlimited

<!ELEMENT parameter (enumeration*)>

<!ATTLIST parameter

label CDATA #REQUIRED

type (int|double|boolean|string|text|color)

id IDREF #REQUIRED

required (true | false) "false"

description CDATA #IMPLIED

default CDATA #IMPLIED

max CDATA #IMPLIED

min CDATA #IMPLIED

encodeForHtml (true | false) "false"

linkTo CDATA #IMPLIED

>

Additional parameters that influence the appearance of the gallery but are not covered by the common parameters defined in the gallery datamodel


  • label - The display label of the parameter
  • type - The datatype of the parameter
  • id - ID of the parameter (unique in scope of plug-in)
  • required - Specify true if a string value must not be empty
  • description - A textual description of the parameter
  • default - Default value for optional parameters
  • max - Only used for numeriic values. Specifies a maximum value.
  • min - Only used for numeriic values. Specifies a minimum value.
  • encodeForHtml - Set to true to encode the label for HTML. Not all web engines - in particular the Flash based engines - support HTML markup in texts.
  • linkTo - Links the values of this parameter to specific values of the target parameter. The target parameter must be a string enumeration.

<!ELEMENT enumeration EMPTY>

<!ATTLIST enumeration

id IDREF #REQUIRED

label CDATA #REQUIRED

>

Enumeration value for string parameters


  • id - ID of enumeration value (unique in scope of parameter)
  • label - Translatable label of enumeration value
