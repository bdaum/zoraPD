Automatic Actions Preferences

Image relations

  • Automatically derive relationships between raw and developed image. When importing both RAW and converted DNG files, it is possible to connect both with a relationship (Original <-> Derivative) which can be viewed in the Hierarchy view. Both the original and the derivate must have the same file name (without extension). You can restrict this detection to images contained in the same folder or allow detection across folder boundaries.

  • Automatically detect derived images by EXIF data. This option allows detecting relationships between images even if the images are not in the RAW or DNG format. The creation date of the photo and other vital image data are used for detecting a relationship.

  • Apply original XMP data to derivates, too. This option allows applying the XMP data of the original image to the derivate image if the derivate image does not have an own XMP file.

Automatic creation of collections

  • Rules for the automatic generation of collections. By adding rules to this table, you can automatically create collections in the catalog when you import images. For each image, the metadata is analyzed. If a rule applies, a collection will be created that will contain the imported image (and others with matching metadata) - if such a collection does not yet exist.
    This table shows an overview of all defined rules. You can edit or remove the rules or add new rules. Details about the rule definition can be found here.

  • Acceleration candidates. It makes sense to speed up the fields specified in the rules by setting up an index in the catalog database. Database indexing (not to be mistaken as text and similarity indexing) significantly speeds up queries on these fields, while slightly slowing down storage and deletion and increasing catalog size. Changes in this table become effective after reopening of the catalog.

      To index a field in the database, check it in the table.
