Collection dialog


The collections dialog is used for defining the query that selects the images of a collection. In addition one or several sort criteria can be defined to sort the results of the query.

General

  • The name of the collection. Adhoc collections do not have a name.
  • A collection description. Adhoc collections do not have a name.
  • A color code to be displayed with the collection in the Catalog view. Adhoc collections do not have a color code.
  • For adhoc-queriers, you may opt to execute the query within the currently collection.
  • In case of a peer-to-peer network you may opt to execute the query across the whole network.

Query

Each query can be made up from several criteria that can be connected via AND and OR operators. The processing sequence is strictly top down  - criteria defined first are evaluated first. There is no operator precedence between AND and OR.
Each criterion consists of a field, a relation and a value. In case of the between and not between relations two values (from and to) must be given. The fields are ordered into three groups: Properties, EXIF and IPTC.

The relations depend on the field type:
  • types with multiple numeric values allow only for the relations = and ≠. In case of = at least one matching value leades to success, in case of ≠ no matching value are allowed.
  • between and not between can only be used with numeric and date types.
  • similar can be used with some numeric and date fields. The tolerances applied during such a query can be configured under.  Window>Preferences>Metadata>Tolerances*.
    Applied text fields, similar performs a case insensitive search.
  • the relations contains and does not contain are also possible with string types meaning that the specified value must be a substring of the field value or not.
  • the relations Wildcard pattern and exclude pattern allow the use of wildcards (*, ?) in the search value.
Sorting
Sort criteria can be connected via AND operators. Fields with multiple values cannot be used in sort criteria. The sort criteria specifed first are the dominant sort critera

Appearance

  • Thumbnail labels. Here you can define a policy how thumbnail labels of images in this collection shall be displayed. If you choose "inherited", the settings of the parent collection, the parent group or of the global preferences are used.
    Select "custom" to define specific templates, font sizes, and alignments for that collection. Note, that subcollections may overwrite those policies.
