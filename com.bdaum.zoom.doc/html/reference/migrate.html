Catalog migration

Catalog migration is required when you want to move catalog and images to another platform - let's say from Windows to Linux. Because the syntax of file paths on both platforms is different (see also Files and Volumes), and also the target folder structure may be significantly different from the source folder structure, more or less extensive transformation of file paths will be necessary.

This affects the following entries in a catalog:

  • Catalog properties
    • Backup location
  • Watched folders
    • Watched folder location
    • Destination folder location (in the case of transfer folders)
    • Filter expressions
  • Images
    • File URI and volume
    • Voice note URI and volume
  • Collections
    • Search values when searching for file URI
  • Exhibitions
    • Destination folder
  • Web galleries
    • Destination folder

The transformation are defined by a list of rules. Each file part to be transformed is compared with the source pattern of each rule, top down. The first rule that matches performs the transformation. Capture groups in the source and the target patterns are used to transport contents from the original file path to the transformed file path.
An empty target pattern indicates that the matching file path corresponds to a file or folder not ported to the target platform.

Windows users please note, that before applying the transformation rules the drive letter in the file path is replaced by the volume name, if any.
Example:

Windows:\\Users\\berth\\Pictures\\raw\\temp\\.*
Windows:\\Users\\berth\\Pictures\\raw\\(.*)\.dng /home/images/dng/$1.dng
Windows:\\Users\\berth\\Pictures\\raw\\(.*) /home/images/$1

        
Here, the content of folder C:\Users\berth\Pictures\raw\ is ported to Linux folder /home/images/ with the exception of subfolder temp\ which is not ported. In addition all DNG-files are placed in a separate subfolder dng/.

The following functions are available to define the list of patterns:
  • Add pattern. Both source and target patterns can be defined freely. The new rule is added to the end of the list.
  • Add folder. First select a folder from the file system. Then the target pattern can be defined - the source pattern is derived from the folder path. The new rule is added to the end of the list.
  • Edit rule. Edit the selected rule.
  • Remove rule. Removes the selected rule from the list.
  • Up. Moves the selected rule one step up.
  • Down. Moves the selected rule one step down.
In addition to these transformation rules you can select a policy how to transform file separation characters (/ and \). For example, when migrating from Windows to Linux you may want to convert the character \ into the character /.
Rules and the policy for file separation characters constitute a migration policy. You can store migration policies under indiviual names in the catalog for later use, or load a migration policy from the list of stored policies.
