Catalog view

The Catalog view shows the image-related information stored in the catalog. This information is organized into different categories:

  • Collections. Collections are defined by a Query that selects certain images from the catalog. Such queries can be defined by the user, but others are already predefined by the system. While user defined collections can be modified freely, system defined collections allow only for the modification of their sort criteria.
    Collections can be nested: you can add sub selections () to any given collection, to refine its query even further. For example: By adding the sub query Exif f-number > 5.6 to the 5-star rating collection, you will create a collection containing all 5-star images exposed with an f-number greater 5.6.
      Sub queriescan be added to both user defined and system defined collections. These are the different collection types:
    • Imports (). Collections of this type are created automatically when importing images. They show the images of previous import operations. Collections of this type can be removed from the catalog, except the Last Import collection ()  that also is displayed in a different way.
    • Categories (). Collections of this type are created automatically - one for each defined category. Categories can be added or deleted freely in Category section of the Properties dialog (Catalog>Properties...>Categories).
    • Ratings (). Collections of this type are created automatically when a new catalog is created. One collection is created for each star-rating and one for non-rated images. Collections of this type cannot be deleted by the user.
    • Time line (). Collections of this type are created automatically when importing images. They group images by their creation date. You can configure the time line in the Catalog Properties (Catalog>Properties...>General). Collections of this type cannot be deleted by the user but are automatically deleted when they become empty.
    • Locations (). Collections of this type can be created automatically when importing images. They group images by their location. You can configure this location collections in the Catalog Properties (Catalog>Properties...>General). Collections of this type cannot be deleted by the user but are automatically deleted when they become empty.
    • Directories (). Collections of this type are created automatically when importing images. They reflect the actual location of the images in the file system. Note, that also the content of removable media is displayed, even if they are not online. To identify images unambiguously, the drive letter is replaced by the volume label. Collections of this type cannot be deleted by the user but are automatically deleted when they become empty.
    • User defined collections (). Collections of this type are exclusively created by the user - either directly in the Catalog view using the context function Create new Collection, or with the help of the gallery menu function Save Query as Collection. For defining arbitrary complex queries most of the meta data fields can be used. Collections of this type may be removed from the catalog by the user.
  • Albums (). Albums are similar to collections but they are not defined through queries. Instead, images are simply pulled from a gallery into the album icon. To create a new album, use the Catalog context function Create new Album. To add a sub-album to an existing album use the Catalog context function Add sub-album.
  • Presentations
    • Exhibitions (). Exhibitions contain the images of real-world exhibitions They help planning and layouting the exhibition design.
    • Slide shows (). Slide shows contain images series to be displayed as a full-screen slide show. In particular, slide shows allow arranging the image sequence and defining time intervals and fading effects.
    • Web galleries (). Web galleries support the compilation and configuration of galleries that are shown within a web site. Web galleries can generate a complete web site that will be presented with the help of a web engine such as Galleriffic, Image Gallery, or Pirobox.

All elements of those categories (collections, albums, presentations) must belong to a named group (). Such groups can be created with the Catalog menu function Create new Group. Each group may only contain elements of the same category.

Menus and actions


The following operations can be executed::
  • Context functions
    • Add subgroup. Adds a nested subgroup to an existing group. This function is only available within user defined groups.
    • Create new Collection. Creates a new user defined collection in the selected group.
    • Add sub-selection. Adds a sub query to the selected collection.
    • Create new Album. This will create a new album in the selected group.
    • Add sub-album. Adds a sub-album to the selected album.
    • Create new presentation>
      • Exhibition. This will create a new exhibition in the selected group.
      • Slideshow. This will create a new slide show in the selected group.
      • Web gallery.This will create a new web gallery in the selected group.
    • Play slideshow. Start to play a slide show presentation. The player is interactive and allows changing image properties while playing.
    • Select all images. Selects all images in the currently opened galleries.
    • Edit. Allows to edit the selected item.
    • Cut. Removes the selected item and places it on the clipboard. No images are removed from the catalog during this process.
    • Copy. Copies the selected item to the clipboard.
    • Paste. Pastes an item from the clipboard into the currently selected item.
    • Delete. Removes the selected item. No images are removed from the catalog during this process.
    • Go into. Reduces the Catalog view to the contents of the currently selected item.
    • Go back. Returns to the previously shown content.
    • Go home. Shows the full catalog.
  • View actions
    • Create new Group (). Defines a new group for collections, albums, or presentations. The first element placed into the group decides about the type of the group.
    • Edit (). Allows to edit the selected item.
    • Delete (). Removes the selected item. No images are removed from the catalog during this process.
    • Properties () . Shows the Catalog Properties.
    • Filter... (). Invokes the Filter dialog for hiding unused or unwanted catalog entries and/or for showing only collections with the selected color code.
    • Collapse all (). Collapses all entries in the Catalog view so that only group entries remain visible.
  • Mouse gestures
    • Single click. Selects an item and show its images in a gallery or a presentation editor.
    • CTRL+Single click. Invokes the edit function on the clicked item.
    • Double click. Expands or collapses the clicked item.
    • Drag. Move items between user defined groups.
