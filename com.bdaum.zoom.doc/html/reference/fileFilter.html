File filter

The list of filter expressions in this dialog determines which files (or subfolders) are accepted and which are not. When a file is tested with the filter, the filter patterns are compared to the file name one by one beginning at the top of the list. When a file name matches with the pattern, the file is either rejected or accepted, depending on the filter action. If the filter expression does not match, the file will be compared with the next filter pattern in the list, and so on. If none of the filter patterns match, the file is accepted. FIlter patterns can be targeted on individual files or on whole folders. This allows to accept or reject subfolders within a directory structure.
Filter expressions may contain wildcards:
  • '*' matches with any string of arbitrary length
  • '?' matches with one and only one character
The filter works case-independent.

Normally, the filter works only on the file or folder name. You may, however, also specify path patterns, such as */prod/*/final/*.* Always use the slash as a path separator not the backslash.
Note: The path pattern match is applied to the relative path of a file or folder, i.e. relative to some base folder such as a watched folder.
