CSS style sheets

The color scheme of the graphical user interface is controlled by CSS style sheets. There are five predefined style sheet names:

p Native Platform color scheme
b Black
d Dark gray
g Gray
w White

Additional color schemes can be added by placing additional CSS style sheets into the dropins/ subfolder within the installation folder. It is also possible to override the built-in style sheets by placing style sheets b.css, d.css, g.css or w.css into the subfolder dropins/ of the installation folder.

Here is an example for the d.css style sheet for Linux platforms. The different GUI widgets are simply addressed with their class names (SWT and Piccolo-SWT GUI elements are used). "*" denotes all classes. Special widgets are addressed by adding a lattice followed by the ID of the widget. In some cases, different settings are defined depending on the style and state information associated with a widget. This is defined in brackets, e.g.[enabled=false]. When searching for a specific color, the first matching rule applies.
