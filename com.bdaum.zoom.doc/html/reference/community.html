Community

Identifier:
com.bdaum.zoom.net.communities.community


Since:
0.9.0

Description:
Extension point for adding new web photo community uploaders


Configuration Markup:


<!ELEMENT extension (community)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>

Extension point for supporting additional web communities



<!ELEMENT account (accountType* , accessType*)>

<!ATTLIST account

authentification (password|authorization)

>

Community account


  • authentification - Authorization type (password, authorization token)

<!ELEMENT community (account)>

<!ATTLIST community

name                  CDATA #REQUIRED

home                  CDATA #REQUIRED

icon                  CDATA #IMPLIED

id                    IDREF #REQUIRED

api                   CDATA #REQUIRED

bannerImage           CDATA #IMPLIED

supportsGeocoding     (true | false) "false"

supportsPhotosets     (false|true|single) "false"

supportsTagging       (true | false) "false"

supportsCategories    (true|false|set) "false"

supportsSubCategories (true|false|set) "false"

sessions              (true | false) "false"

>

Web photo community


  • name - Translatable name of community
  • home - Home URL
  • icon - Community logo
  • id - Unique community ID
  • api - Class implementing the image uploader. Must extend org.scohen.juploadr.uploadapi.ImageUploadApi
  • bannerImage - Larger community logo for use in wizard banner, approx. 64x64 pixels
  • supportsGeocoding - Specifies if community supports geocoding
  • supportsPhotosets - false: No photosets at all true: nor or multiple photosets per image single: exactly one photoset per image
  • supportsTagging - Specifies if community supports tagging (keywords)
  • supportsCategories - false: no categories at all true: images may declare a category set: only photosets may declare a category
  • supportsSubCategories - false: no subcategories at all true: images may declare a subcategory set: only photosets may declare a subcategory
  • sessions - Specifies if community needs session logic

<!ELEMENT accessType (accessTypeDetail*)>

<!ATTLIST accessType

key       CDATA #REQUIRED

label     CDATA #REQUIRED

isDefault (true | false) "false"

>

access type such as private or public


  • key - Unique key
  • label - Translatable label
  • isDefault - true, if this is the default access type

<!ELEMENT accountType EMPTY>

<!ATTLIST accountType

key   CDATA #REQUIRED

label CDATA #REQUIRED

>

A possible account type such as free or pro.


  • key - Unique key
  • label - Translatable label

<!ELEMENT accessTypeDetail EMPTY>

<!ATTLIST accessTypeDetail

key       CDATA #REQUIRED

label     CDATA #REQUIRED

initValue (true | false) "false"

>

Access type detail information


  • key - Unique key
  • label - Translatable label
  • initValue - Initial value
