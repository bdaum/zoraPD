Rules for automatic creation of collections

Group

This attribute is for convenience only for restricting the number of displayed fields. If in doubt select All.

Field

Each rule must refer to exactly one metadata field such as the EXIF field for exposure time or the IPTC field for model age. Not all existing metadata fields are offered here, in particular fields that are already used for system collections (such as the IPTC creation date that is reflected in the timeline collections) are not available for the definition of rules.

Type

The type of a rule is derived from the metadata field selected:
  • Discrete. The field only has a few possible values such as the EXIF field flash_fired.
  • Linear. Field values are distributed in a linear fashion. It makes sense to create collections with value ranges equal in size. This type can also cover negative values. A typical example is the field EXIF field for light_value.
  • Exponential. Field values are distributed exponentially. It makes sense to create collections with value ranges steadily growing in size, so that collections can cover both very small values and very large values. A typical example is the EXIF field for exposure time. This type cannot cover negative values.

Parameter

  • Rules with discrete type do not have parameters.
  • For rules with linear type you can specify the with of the equidistant value ranges. For example, if you specify an interval size of 2 for the EXIF field light_value, the light values 0 and 1 will be grouped into one collection, the values 2 and 3 into another collection, and so on.
    Alternatively, you may specify intervals explicitely as a semicolon separated list such as 0; 2; 5; 8; 16
  • For rules with exponential type you can specify the base value of an exponential series. For example, if you specify 2 as base value, this would result in an interval series such as ...; 0.25; 0.5; 1; 2; 4; 8; ....  Accordingly collections would be created with value ranges of 0.25 - 0.5; 0.5 - 1; 1 - 2; 2-4; 4-8 and so on.
    Alternatively, you may specify intervals explicitely as a semicolon separated list such as 0; 2; 5; 8; 16.
