Web Gallery designer view

The purpose of the Web Gallery designer view is the design of photographic exhibitions to be published on the World Wide Web. The images can be arranged in sequence in different storyboards before the gallery is generated. Simply place images on the storyboards by dragging them from a gallery to a storyboard. By pressing Ctrl when dragging images over an existing storyboard image the new images will replace the storyboard image. Otherwise they are simply added to the storyboard.

Per drag & drop you may rearrange the images freely - even between storyboards. Images can be deleted by dragging them over the top border of the view.

Each image is equipped with a caption below the image. All captions are editable. Simply click an entry to edit it. You may also edit the image properties (including the caption) by clicking the green Properties button at the left side of the image caption.

Local menu functions and tool buttons
  • Add storyboard (). Adds a new storyboard to the exhibition and shows a dialog with the Storyboard Properties. Each storyboard contains two buttons in the lower left corner: one to delete the storyboard, and one to edit the storyboard properties.
  • Web Gallery Properties (). Shows the dialog with the web gallery properties.
  • Synchronize (). Toggles synchronization mode. If this mode is set the selected image will be shown in the current collection (if present), too.
  • Generate (). Generates the gallery and shows it in the platforms web browser.
  • Save as Design... (). Generates the gallery and shows it in a dialog. The appearance settings and other design related settings can be saved under a name. Such a design can later be loaded into an existing or new web gallery.

Navigation

Navigation is similar to the Sleeves view. Just drag the whole area to pan. Make sure that the cursor is not positioned on an image when starting dragging. Ctrl accelerates the panning operation. All other modification keys (except the zoom key, see below) can be used to force panning, even is the cursor is positioned on an image.

You can adjust the viewing distance by pressing the zoom key (ALT or Shift key or the right mouse button depending on the preference settings) and dragging left or right.Alternatively, you may zoom using the mouse wheel.

A double click on the exhibit opens the properties dialog.

Web Gallery properties

  • General
    • Name. The gallery name.
    • Description. Short description of the web gallery. This field may contain HTML mark-up.
    • Name of HTML page. This is the name of the entry web page into the generated web gallery. The default is "index.html".
    • Keywords. Specify keywords to be inserted into the header of the generated entry web page.
    • Download link. Specify a text such as "Download Original". This link will be shown within the captions of an image and allow the user downloading the original file. If you leave this field empty, no download link will be generated. Please remember that not all users might be able to process or view the format of your originals - especially when they are in the RAW or DNG format. Note also, that your originals may contain meta data which will become visible to the public.
    • Name plate. This can be a GIF-, JPEG-, or PNG-image that will be placed at the very top of the page.
    • Create watermarks. Adds a copyright notice to each single picture in form of a watermark. The copyright property of the image will be used for the watermark. If the image does not specify an own copyright, the gallery copyright (below) is used instead. 
    • Copyright. Please specify a copyright remark here (the copyright that applies to the complete gallery, not to the single images). The copyright remark will be inserted into the footer of the generated web page.
    • Contact. A contact name that will be displayed in the footer of the generated web page.
    • Email. An email address for contact. The contact name will be equipped with an email link to that address.
    • Web URL. A web URL that will be displayed in the footer of the generated plate. If specified, also the name plate will be equipped with a link to that page.
    • Engine Reference. Generates the specified text followed by the Engine name as last line on the page. A double click on the input field will insert the text "Powered by".
  • Appearance
    • Background image. This can be a GIF-, JPEG-, or PNG-image that will be shown in the background of the generated page.
    • repeat. Check this box if you want a small background image to repeat to fill the available space.
    • Background color. The color of the background if no image is defined
    • Shaded area color. The color of shaded areas, if any.
    • Border color. The color of border lines.
    • Link color. The color of clickable links.
    • Thumbnail size. Select the desired thumbnail sizes. If a square size is selected, the center part of the image will be used.
    • Opacity. The opacity of thumbnails. Some web engines use this to highlight the currently selected thumbnail.
    • Padding. The space between thumbnails in pixels.
    • Margins. Page margins at top, right, bottom and left in pixels.
    • Title font. Font and color of the page title.
    • Subtitle font. Font and color of subtitles. If the web engine supports multiple storyboards, the storyboard title is formatted with this font.
    • Caption font. Font and color of image titles.
    • Navigation font. Font and color of navigation hyper-links.
    • Description font. Font and color of standard text.
    • Footer font. Font and color of the page footer.
  • HTML
    • In <HEAD> element. Define HTML snippets to be inserted into the <HEAD> element such as <SCRIPT> or <LINK> sub elements.
    • Top HTML. Define HTML snippets to be inserted directly after the <BODY> tag.
    • Bottom HTML. Define HTML snippets to be inserted directly before the </BODY> end tag.
  • Meta data
    • This group shows selectable EXIF- and IPTC-properties that can be included into the generated images. Thumbnails will never contain meta data. Uploaded originals are not affected by these settings - they are always uploaded with their original meta data.
    • Show metadata in captions. Check this mark to not only include the meta data into the images but to make the selected meta data fields visible within the image captions.
  • Web Engine
    • Engine. A variety of web engines can be selected. Each engine may require additional special parameters which are listed in the lower part of the dialog page.
  • Output
    • Local folder. Please select an output folder for the generated web gallery if the output is to be created locally.
    • FTP directory. Please select an existing FTP account or create a new one if you want to upload the output via FTP.
    • You can influence the Output Quality with these setttings.

Storyboard properties

  • Title. The storyboard title will be displayed as a subtitle if the web engine supports multiple storyboards. May remain empty.
  • Description. Short description of the storyboard. This field may contain HTML mark-up.
  • Show captions. Check this box if you want to show image captions for the images of that storyboard.
  • Show descriptions. Check this box if you want to show image descriptions for the images of that storyboard.
  • Show EXIF data. Check this box if you want to show EXIF data for the images of that storyboard.
  • Image size. Select a suitable image size for all images of that storyboard.
  • Enlarge small images. Check this box if small images should be enlarged to the selected image size. Large images will always be reduced to the selected image size.

Image properties

  • Caption. The image title to be displayed as an image caption.
  • Description. Short description of the image that will be displayed in the caption area of the image but with a different font (Description Font). This field may contain HTML mark-up.
  • HTML Alt-Text. Text to be inserted into the <ALT>-Element of the web page. This text will be displayed while the image is loading or when it can't be loaded.
  • Downloadable. Check this box, if you want a download link for this image. In this case, the original image, too, is copied into the output folder.
  • Include Metadata. Check this box if you want to include meta data in this image. This will also affect the display of meta data in the caption area.

Context menu

To obtain the context menu, right click an image.
  • Jump to image. Jumps to an image selected from a list.
  • View image fullscreen. Shows the image in full resolution.
  • Modify. Edit or rotate the image.
  • Show folder. Shows the selected image in its file system folder.
  • Show in timeline. Shows the selected image in its timeline collection.
  • Show in map. Shows the selected image in the map view  (only with geo-coded images)
  • Play voice note. Plays the voicenote attached to the image, if any.
  • Edit storyboard properties. Allows to edit the properties of the selected storyboard.
  • Delete storyboard. Deletes the complete storyboard.
