Tuning


There are several places where you can tweak the system for higher speed:

Start-up

For a faster start-up you may want to consider excluding the folder Programs/ZoRaPD from your Antivirus application after running a virus check across this folder.

Display

Window>Preferences>General*
  • Display
    • Output ICC-profile. Switch to Disable CMS to speed up the display of images and thumbnails
    • Use quality interpolation. Switch this option off for faster display of images and thumbnails. Slight pixelation of images may occur.
    • Precede high-res image with preview image in internal viewer where possible. Switch this option off for faster full screen image viewing.
    • Hide thumbnail JPeG artifacts in internal image viewer. Switch this option off for faster full screen image viewing.

Window>Preferences>Appearance*
  • Color scheme.Choose platform for fastest rendering of views and dialogs.

Import

Catalog>Properties...

  • General. 
    • Automatically create timeline. If you are not interested in a timeline, switch these options off to speed up the import process.
    • Automatically create location collections. If you are not interested in location collections, switch these options off to speed up the import process.
  • Thumbnails
    • Resolution. Keep the thumbnail size small or medium to speed up the import process.
    • Derive thumbnails from preview images. Checking this option will speed up the import of most RAW images considerably. The thumbnail may, however, differ somewhat from the high-res image.
    • Sharpen during import. For maximum import speed disable thumbnail sharpening.


Window>Preferences>Import*.

General

  • Relationships
    • Automatically detect derived images by EXIF data. Switch this option off if you are not interested in relationships between images.

RAW conversion

Raw conversion is performed by the DCRAW module. You can speed this process up by choosing the appropriate DCRAW parameters (see below) but also by replacing the built-in DCRAW module with an optimized version. For example, on Windows the built-in DCRAW module handles all Windows versions from XP onwards. The version optimized for Vista will run somewhat faster. Just search the Internet for such a special built, install it on your harddisk, and specify its path under Window>Preferences>Import>RAW Conversion*).
  • Options
    • Interpolation. Choose bilinear interpolation for speeding up RAW conversion.
    • Iterations. Set the number if iterations to 0 for speeding up RAW conversion.
    • Highlight recovery. Switch this option off for speeding up RAW conversion.
  • Recipe Detectors. Select only the RAW converters that you are actually using.
Raw Therapee 2.4.x may cause substantial performance degradation because it will rewrite all recipes in the image folder causing ZoRa Photo Director to re-import all associated images. Raw Therapee 3.0.x that does not expose this behavior. We therefore strongly recommend usingRaw Therapee 3.0.x.
    • Process recipes when importing RAW images. Unchecking this option will speed up operation especially when editing images - provided the image folder belongs to the watched folders.

DNG conversion

  • Relationships
    • Automatically derive relationships between raw and developed image. Switch this option off if you are not interested in relationships between images.

Indexing

There are two areas where indexing takes place:
  • Metadata indexing. Some metadata fields are indexed in the catalog database (Db4o). Indexing speeds up the query process considerably: A query for a non-indexed metadata field will scan through all images, while a query for an indexed metadata field will find matching images rather quickly. Of course, maintaining an index will take some time, so imports and deletions will be slower when many metadata fields are indexed. Also the catalog becomes larger. You can configure metadata indexing under Window>Preferences>Metadata>Tuning*).
  • Text search and similarity. For text and similarity searches a separate indexing process is used (Lucene). This indexing is always performed in the background. Speeding up this process will improve general performance. You can configure this indexing process under Catalog>Properties>Indexing. You may want to reduce the number of selected similarity algorithms to a minimum because each algorithm requires extra steps when constructing the index. On low level platforms you may want to switch indexing off, altogether, or even exclude the indexing module during installation.

General operation

Catalog>Properties...

  • Watched folders. Keep the number of watched folders to a minimum. This will reduce background activity and improve overall performance.

  • Defragmentation. Defragmenting the catalog now and then can improve the query performance. Go to Statistics tab page and click the Defragment now link. This can take some minutes.
