Catalog maintenance

The main menu offers several functions for catalog maintenance:

  • Catalog>Maintenance...>Split catalog...  allows you transferring a selection of images into a new catalog and to remove those images from the current catalog. 
    Example: The current catalog contains images from several photographers. You want to split the catalog by photographer. Start an adhoc search (Metadata>Adhoc search...) for IPTC>Artist. Then call Catalog>Maintenance...>Split catalog... You may also split a subset of the images shown by selecting single images.
  • Catalog>Maintenance...>Merge catalogs... allows you combining two catalogs into one. Several options are provided how to handle duplicates.
  • Catalog>Maintenance...>Recreate index can be used to refresh a catalogs Lucene index. This index is used for similarity search and text retrieval. Only use this function if you have reason to believe that the index is corrupt (wrong search results during similarity search or text retrieval).
  • Catalog>Maintenance...>Restrore... can be used to restore a catalog from a back-up. You may restore even a different catalog from the currently open catalog. After the catalog has been restored, it is opened.
  • Catalog>Maintenance...>Migrate...can be used to migrate a catalog to a diffrerent platform (e.g. Windows to Linux). In the process the folder structures can be changed, too. However, you need some knowledge in regular expressions to use the function.
