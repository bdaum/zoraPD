Importing images

To add images to the catalog you have to import them. You can

  • import single images. Invoke (File>Import...>File ...) and select one or several files. Alternatively you may drag image files into an image area such as the Lightbox view or the Sleeves view. Then you can choose between leaving the images at their original position or importing them into a new folder structure. In the later case the images are copied from the source folder to the target folder. It is possible to rename the imported images during that process.
  • import whole folders. Invoke (File>Import...>Folder ...) and select a folder. Alternatively you may drag a folder into an image area such as the Lightbox view or the Sleeves view. Then you can choose between leaving the images at their original position or importing them into a new folder structure. In the later case the images are copied from the source folder to the target folder. It is possible to rename the imported images during that process.
  • import from a device (File>Import...>From Device ...) such as a Memory Card or a connected Digital Camera. Invoke (File>Import from Device ...). After selecting the images to import and specifying a target folder, the images are copied from the device to the target folder and are imported into the catalog. It is possible to rename the imported images during that process.
  • import analog images (File>Import...>Analog images ...). This function allows you importing images that have been scanned in. Typically, these images do not contain metadata such as EXIF and IPTC data. Therefore, you are prompted with a dialog allowing you to enter vital metadata to be added to the imported images.

Keeping the catalogue up-to-date

To keep your catalog up to date, you have the option (Catalog>Properties...>Watched Folders) to add folders to a list a watched folders. Whenever a new image is added to a watched folder, it is automatically imported into the catalog. Modifications of images are detected, too: a modified image will be re-imported into the catalog. Alternatively, you have the option to synchronize selected images manually (File>Refresh...).

Imported data

During the import of images, a thumbnail and the meta data (EXIF and IPTC) are extracted and added to the catalog. The original images are not required to view the catalog, so you can work with removable media. However, some highres-operations such as Editing, Viewing, Slide show, or Exporting require that these media are mounted.

Remote images

Images that have been uploaded to FTP accounts or to web-sites can be added to the catalog, too, to keep track of them. (This is not possible for images uploaded to photo communities such as flickr because photo communities usually deny direct access to uploaded images.) When required, remote images are downloaded automatically.

Raw images

By default, thumbnail images are derived directly from the high-res original image. RAW- und DNG-images need a conversion which is performed on the fly. Because this may take quite some time, you may opt to derive thumbnails from the preview images contained in the RAW- or DNG-files. This will speed up the import process considerably, but the results are quite different. You may want to try both options for your images before deciding for a specific variant (Window>Preferences>Import*).

Currently, ZoRa Photo Director supports two raw converters: DCRAW and RawTherapee.

  • DCRAW is a program that is normally started from the command line. ZoRa Photo Director integrates DCRAW and calls it up with the appropriate parameters as needed.
    • Windows and MacOS distributions of ZoRa Photo Director come with a built-in version of DCRAW. Newer DCRAW versions can simply be dropped into the subfolder dropins/ of the installation folder.
    • On Linux platforms you should install DCRAW into /usr/bin. You can also install into the sub folder dropins/ or specify a location under Window>Preferences>Import>RAW Conversion. You can find a DCRAW version for your Linux system via  http://linuxappfinder.com/package/dcraw. 
  • Alternatively, you can use RawTherapee for raw conversion. This must be installed separately. You must specify the location of the RawTherapee executable under Window>Preferences>Import>RAW Conversion*.

DNG

DNG files can automatically be derived when importing RAW images to allow for an integrated RAW work flow.
Before doing so, please set the location of your DNG converter (Window>Preferences>Import>DNG Conversion*). You can download a copy of the free Adobe DNG Converter from www.adobe.com/products/dng/.
When deriving DNG files you have the option, to embed the original RAW file into the DNG file (The Adobe DNG Converter can extract the original RAW file from such a DNG file), to import both RAW- and DNG-file side by side (default), or to import only the DNG version.

Note: This functionality is not available under Linux, because there is no Linux-version of the Adobe DNG converter.

Raw Recipe Reflection (3R)

Raw and DNG images are typically processed with a RAW converter such as Adobe Camera Raw, Adobe Lightroom, Apple Aperture, Capture One, DxO, Helicon Filter, RawTherapee, and others. The corrections that you make in those converters are, however, not applied to the raw image file, but are stored in a separate and fairly small metafiles (recipe). As a matter of fact  each raw converter sports its own recipe format and its own way of storing this file.

The ZoRa Photo Director is able to detect the recipes of some raw converters and to re-apply the development parameters to raw images as they are imported or displayed. You should, however, not expect a perfect match with the image developed with the original raw converter. The aim is to provide an approximate visual impression how the raw image will look. In particular, ZoRa Photo Director does currently not re-apply partial corrections or the correction of distortions.

This recipe detection and processing can be configured under Window>Preferences>Import>RAW Conversion*.


If a recipe is found for an image, this is reflected in the status of the image:

RAW image without recipe: RAW
DNG image without recipe: Converted
RAW or DNG image with recipe: Corrected

Because one can build search queries on the status of image, you can easily separate between non-corrected and corrected RAW images.
