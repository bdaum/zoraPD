Acknowledgements

The application contains several Open Source components that substantially contribute to its functionality:

  • The Eclipse RCP platform. This is the main enabling framework implementing workbench and views (www.eclipse.org).
  • The Tag Cloud UI is based on the Zest Cloudio (http://www.eclipse.org/gef/zest/) component from Stephan Schwiebert.
  • Peer-to-Peer communication is based on R-OSGi from Jan S. Rellermeyer.
  • MTP support is based on jMTPe by Pieter De Rycke, Philipp Ebert and Martin Heinzerling
  • db4o is used as database engine for the catalog (www.db4o.com).
  • The Lightbox gallery is based on the Eclipse Nebula project (www.eclipse.org/nebula).
  • The CSS engine is provided by the TK-UI project (http://tk-ui.sourceforge.net/index.html).
  • The Sleeves gallery, the slide show, exhibition, and web gallery designers are based on the piccolo2D framework from the University of Maryland (www.piccolo2d.org).
  • Meta data extraction is done with the help of Phil Harveys well-known ExifTool (www.sno.phy.queensu.ca/~phil/exiftool).
  • Import from XMP files and export to XMP files is based on the Adobe XMP toolkit (www.adobe.com/devnet/xmp).
  • Some image formats are read with the help of ImageJ (http://rsbweb.nih.gov/ij).
  • Advanced image subsampling by Morten Nobel (http://blog.nobel-joergensen.com).
  • RAW processing is performed with the help of dcraw by Dave Coffin (www.cybercom.net/~dcoffin/dcraw).
  • EXR processing is performed with the help of exrForJava by Mario Fichtenmayer (www.dampf-fabrik.de).
  • Radiance HDR and PFM processing is performed with the help of the ForkLabs-HDR-codec (https://forklabs.dev.java.net).
  • Java ImageIO extension by Harald Kuhr (https://github.com/haraldk/TwelveMonkeys)
  • Video support based on ffmpeg.
  • JavaCV is used to drive ffmpeg (http://bytedeco.org/).
  • Media ejection is performed with the help of EjectMedia by Uwe Sieber (www.uwe-sieber.de/drivetools.html)
  • Online geo naming relies on the webservice provided by www.geonames.org. (www.geonames.org).
  • The Google Map API and the Google Map Webservice is used for showing geographic locations (http://code.google.com/intl/en/apis/maps/).
  • The Microsoft Bing Maps API is used alternatively for showing geographic locations (http://msdn.microsoft.com/en-us/library/bb429619.aspx).
  • The markers in the maps are created with the help of the Map Icon Makers by Pamela Fox. Markers are managed using the MarkerClusterers by Xiaoxi Wu and Luke Mahe.
  • Various map services like OpenStreetMap or esri maps are accessed via the Leaflet API from Vladimir Agafonkin (http://leafletjs.com)
  • The MediaUtil by Dmitriy Rogatkin and Suresh Mahalingam is used for rotating thumbnails loslessly. (http://mediachest.sourceforge.net/mediautil).
  • The jdic project provides the integration with the platform's email client (https://jdic.dev.java.net).
  • iText is used for generating PDF output. (www.lowagie.com/iText).
  • Lire from SemanticMetadata.net is used as the component for Content based Image Retrieval (www.semanticmetadata.net/lire).
  • Apache Lucene is used for full text search (http://lucene.apache.org).
  • jUploadr is used for uploading to locr and smugmug (http://juploadr.org/)
  • Scribe is used for OAuth authorization to web communities (https://github.com/fernandezpablo85/scribe-java).
  • Flickr4Java (https://github.com/callmeal/Flickr4Java) is used for uploading to flickr.
  • The Gallerific web-engine is produced by Trent Foley (www.twospy.com/galleriffic).
  • The Image Gallery web-engine is produced by Tony Yoo (http://tonyyoo.com).
  • The Pirobox web-engine is produced by Pirolab Webdesign (http://www.pirolab.it).
  • The Galleria web-engine is produced by Aino (http://www.aino.se/).
  • The SimpleViewer and TiltViewer web-engines are produces by SimpleViewer Inc. (www.simpleviewer.net).
  • The 3D-walk-through online exhibition is based on work published by The National Museum of China (http://www.visitnmc.com/) and Gerard Fernandez (www.dhteumeuleu.com) 
  • FTP support is based on Apache Commons Net (http://commons.apache.org/net/)
  • Hunspell Spell Checker by László Németh (http://hunspell.sourceforge.net/);
  • Java binding for Hunspell by Flemming Frandsen (http://dren.dk/hunspell.html)
  • Java Native Access (JNA) by Timothy Wall and others. 
  • SQLite foreign catalog access David Crawshow https://bitbucket.org/xerial/sqlite-jdbc
  • Printing support by the PaperClips project from Matthew Hall (http://code.google.com/p/swt-paperclips/)
  • Jama from The MathWorks and the National Institute of Standards and Technology (NIST) is used for matrix algebra. (http://math.nist.gov/javanumerics/jama/).
  • Full screen mode by Ugo Sangiorgi and others (http://ugosan.org/).
  • Charting by David Gilbert et al (http://www.jfree.org/)
  • The webserver is based on the HttpServer by Amichai Rothman (http://www.freeutils.net/source/jlhttp/).
  • Dropzone in web pages by Matias Meno (http://www.dropzonejs.com)
  • QR code: Project Nayuki (https://www.nayuki.io/page/qr-code-generator-library)
  • Most of the icons were created by Mark James (http://www.famfamfam.com/lab/icons/silk/) and Damien Guard (http://www.damieng.com/icons/silkcompanion)
  • Thanks to Johannes Haberl for creative input
  • Logo design by Alice Philipp.
