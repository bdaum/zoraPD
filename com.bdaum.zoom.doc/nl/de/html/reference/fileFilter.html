Dateifilter

Die Liste der Filter-Ausdrücke in diesem Dialog legt fest, welche Dateien akzeptiert werden und welche nicht. Wenn eine Datei mit dem Filter getestet wird, werden die Filter-Muster mit den Dateinamen einer nach anderem vom der Liste an verglichen. Wenn ein Dateiname mit dem Muster übereinstimmt, wird die Datei entweder abgelehnt oder angenommen, je nach Filter-Aktion. Wenn der Filter-Ausdruck nicht übereinstimmt, wird die Datei mit dem nächsten Filter-Muster in der Liste verglichen und so weiter. Wenn keines der Filter-Muster übereinstimmt wird die Datei akzeptiert. FIlter-Muster können auf einzelne Dateien oder ganze Ordner angwendet werden. Dies erlaubt die Annahme oder Ablehnung ganzer Unterordner innerhalb einer Verzeichnis-Struktur.

Filterausdrücke können Platzhaltersymboler (wildcards) enthalten:
  • ' * ' entspricht einer Zeichenfolge beliebiger Länge 
  • '? ' entspricht genau einem Zeichen
Das Filter arbeitet unabhängig von Groß- und Kleinschreibung.

Normalerweise wird das Filter auf den Datei- oder Ordnernamen angewandt. Sie können allerdings auch Pfadmuster spezifizieren, z.B. */prod/*/final/*.* Benutzen Sie dabei immer den Schrägstrich und nicht den umgekehrten Schrägstrich als Pfadtrennzeichen. Wurde ein solches Pfadmuster angegeben, so wird es auf den relativen Pfad einer Datei oder eines Ordners angewandt, d.h. relativ zu einem Basisordner wie z.B. einem überwachten Ordner.
