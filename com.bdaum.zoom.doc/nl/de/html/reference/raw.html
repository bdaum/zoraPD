Working with Raw images

The RAW image format is popular for many serious photographers. The reason is, that other than the limited and lossy JPeG format, RAW files contain all data captured by the cameras sensor.

RAW images cannot directly be used for display or print purposes. Instead, they need to be developed into a traditional image format such as TIFF or JPeG using a Raw Converter, for example Adobe Camera Raw, Adobe Lightroom, Capture One, Raw Therapee, Bibble, LightZone and others.

Unfortunately, each camera manufacturer maintains its own RAW format. In an attempt to standardize the RAW image format, Adobe has created the DNG format (Digital Negative). A few camera manufacturers support this camera format directly, but in most cases the the images delivered by the camera must be converted using Adobe's DNG Converter. Still, these DNG images must be processed with a RAW converter to obtain displayable images. However, not all RAW converters can convert DNG files obtained with Adobe's DNG Converter. So, our advice is to store RAW images both in the RAW format and in the DNG format.

Basic RAW support


ZoRa Photo Director can directly import RAW and DNG images. In the process it can optionally convert RAW images to DNG by integrating the Adobe DNG converter. (Note, that Adobe DNG Converter is not distributed with ZoRa Photo Director but must be downloaded from Adobe and installed separately.)

By default ZoRa Photo Director uses internally the popular module dcraw to convert RAW images into a viewable format (see also Importing images). This is done every time when a thumbnail is produced from an image, when images are viewed or exported. Typically, the conversion process takes a few seconds. Because new digital cameras are released frequently, dcraw versions change frequently - more often than ZoRa PD versions. To support the latest digital cameras, new dcraw versions can therefore simply be dropped into the sub folder dropins/ in the ZoRa Photo Director installation folder.

Alternatively, ZoRa Photo Director can use the RawTherapee raw converter in batch mode instead of dcraw. The quality should be better than with dcraw, and if you dominantly use RawTherapee to convert your raw images, integrating it with ZoRa Photo Director is a good way to keep your image base consistent. Also, on OS X platforms the availability of RawTherapee is better than the availability of dcraw. In addition, RawTherapee offers the possibilty for adding support to new cameras - look here!

To set up RAW and DNG conversion go to Window>Preferences>Import>RAW Conversion*.

ZoRa Photo Director can automatically track the relationships between RAW images and their derived images. You can visualize these relationships through the Hierarchy view.

Raw Recipe Reflection (3R)

When developing RAW images, Raw Converters do not modify the original image. Instead they apply certain transformations to the original image and store the result in a conventional image format such as TIFF or JPeG. These transformations (recipes) are usually stored in separate files, for example in XMP sidecars or in separate folders, so that later these transformations can be reapplied or modified without effort.

ZoRa Photo Director is able to read these recipes and to apply them to the image whenever the image is converted into a viewable format. This is optional and can be configured under Window>Preferences>Import>RAW Conversion*. In case that one image has several recipes created with different Raw Converters, ZoRa Photo Director will alway use the newest recipe. In the rare case that two Raw Converters share the same recipe file (as is the case between Adobe Camera Raw/Adobe Lightroom and BIbble), the priority defined under Window>Preferences>Import>RAW Conversion* decides which recipe will be used.

RAW recipes that were used for deriving other images (such as TIFF or JPeG images) can automatically be archived within the catalog (Window>Preferences>Import>RAW Conversion*). This allows you at a later time, to restore the original recipe. To do so, open the Hierarchy view on the RAW image (Relations>Show Derivatives) and double click the derivative. In the following dialog click Restore.

Note, however, that the above applies only when using dcraw as raw converter. If you use RawTherapee you have only the options to apply .pp3 sidecar recipes or the standard processing profile setup in RawTherapee. Recipes from other raw converters are ignored. The advandage is that the recipe is processed by RawTherapee itself and therefore is 100% identical with RawTherapee conversion results.

Advanced Features: CHDK support

The CHDK is a tool that can be loaded onto an memory card and enables most Canon consumer cameras - among other things - to deliver RAW and DNG files. Unfortunately, this RAW format does not contain EXIF data, and the DNG files only contain an incomplete set of EXIF data. Fortunatly, the camera also produces an associated JPeG file with a complete set of EXIF data.

To obtain the most complete set of metadata with these files, we recommend to work in the following way:
  • Set up the camera to deliver DNG images (CHDK menu). This allow you to process these images with most RAW converters (the RAW converter support for the CHDK propietary RAW format is very limited, indeed.)
  • Set up the camera (CHDK menu) to prefix image names with IMG_.
  • When importing those images from the memory card with the ZoRA Photo Director, check the option Enabling prefix for EXIF transfer under Advanced Options (see also Import from device dialog). ZoRa Photo Director will then import the image from the DNG file but the EXIF data from the associated JPeG file.
