Dateien und Datenträger

Bilder und Videos werden anhand ihrer Dateipfade identifiziert. Da dieser nicht auf allen Plattformen eindeutig ist, wird zusätzlich noch der Name des Datenträgers hinzu gezogen.

So bestehen z.B. auf Windows Dateipfade aus dem Laufwerksbuchstaben, gefolgt von einer Folge von Ordnern und Unterordnern. Das Problem hierbei sind externe Datenträger, denen in der Regel von Windows der selbe Laufwerksbuchstabe zugewiesen wird. Da ZoRa PhotoDirector auch Bilder von externen Datenträgern katalogisieren kann, wird zur Identifizierung von Bildern zusätzlich der Name des Datenträgers (CD, DVD, USB-Stick, SD-Card) heran gezogen.
Es ist deshalb wichtig, solche Datenträger beim Formatieren immer mit einem eindeutigen Namen zu versehen!

Auf Linux und Mac gibt es andere Probleme. Hier ist der Datenträgername Teil des Dateipfades, jedoch weicht die Syntax des Dateipfades von Plattform zu Plattform voneinander ab. So haben Dateipfade von externen Medien auf Ubuntu die Form

/media/{datenträger}/...

auf Suse-System dagegen

/run/media/{benutzer}/{datenträger}/...

auf MacOS wiederum

/Volumes/{datenträger}/...


Die entsprechenden Pfadmuster sind in der Datei volumePatterns.ini in Plugin com.bdaum.zoom.image eingetragen und können bei Bedarf verändert werden. Verwendet werden dabei reguläre Ausdrücke.

Die Standarddefinition dieser Datei ist:

# MAC
/Volumes/(.*?)(?:/.*)?
# UBUNTU
/media/(.*?)(?:/.*)?
# SUSE
/run/media/.*?/(.*?)(?:/.*)?

Es ist noch anzumerken, dass diese Muster auch verwendet werden, wenn Dateien auf Linux- oder Mac-Systeme migriert werden sollen (Katalog>Wartung>Katalog migrieren...).
