Danksagungen


Diese Anwendung enthält mehrere Open-Source-Komponenten, die wesentlich zur Funktionalität beitragen:

  • Die Eclipse RCP-Plattform. Sie stellt das Grundgerüsts der Anwendung mit Werkbank und Sichtfenstern zur Verfügung (www.eclipse.org).
  • Die grafische Darstellung der Tagwolke beruht auf der Zest-Cloudio-Komponente (http://www.eclipse.org/gef/zest/) von Stephan Schwiebert.
  • Die Peer-zu-Peer-Kommunikation basiert auf R-OSGi von Jan S. Rellermeyer.
  • Die MTP-Unterstützung basiert auf jMTPe von Pieter De Rycke, Philipp Ebert und Martin Heinzerling
  • db4o wird als Datenbank-Engine für den Katalog verwendet (www.db4o.com).
  • Die Lichtkasten-Galerie stammt aus dem Eclipse-Nebula-Projekt (www.eclipse.org/nebula).
  • Die CSS-Engine stammt aus dem TK-UI-Project (http://tk-ui.sourceforge.net/index.html).
  • Die Hüllen-Galerie, die Designer für Ausstellungen, Diashow und Webgalerien sind auf der Grundlage von piccolo2D der Universität von Maryland implementiert (www.piccolo2d.org).
  • Die Metadaten-Extraktion erfolgt mit Hilfe von Phil Harveys exiftool (www.sno.phy.queensu.ca/~phil/exiftool).
  • Import und Export von XMP-Dateien basieren auf dem Adobe-XMP-Toolkit (www.adobe.com/devnet/xmp).
  • Einige Bildformate werden mit Hilfe von ImageJ eingelesen (http://rsbweb.nih.gov/ij).
  • Subsampling-Algorithmen von Morten Nobel (http://blog.nobel-joergensen.com).
  • Die RAW-Verarbeitung erfolgt mit Hilfe von dcraw von Dave Coffin (www.cybercom.net/~dcoffin/dcraw).
  • Die EXR-Verarbeitung erfolgt mit Hilfe von exrForJava von Mario Fichtenmayer (www.dampf-fabrik.de).
  • Die Verarbeitung  von Radiance-HDR- und PFM-Dateien erfolgt mit Hilfe des ForkLabs-HDR-Codec von ForkLabs (https://forklabs.dev.java.net).
  • Java-ImageIO-Erweiterungen vonHarald Kuhr (https://github.com/haraldk/TwelveMonkeys)
  • Die Videounterstützung basiert auf ffmpeg.
  • JavaCV wird benutzt, um ffmpeg anzusteuern (http://bytedeco.org/).
  • Der Auswurf von Medien erfolgt mit Hilfe von EjectMedia von Uwe Sieber (www.uwe-sieber.de/drivetools.html)
  • Die Vergabe von Geonamen basiert auf einem Web-Service, der von www.geonames.org zur Verfügung gestellt wird. (www.geonames.org).
  • Das Google Map API und der Google Map Webservice werden für die Anzeige geographischer Positionen verwendet. (http://code.google.com/intl/de-DE/apis/maps/).
  • Das Microsoft Bing Maps API wird alternativ für die Anzeige geographischer Positionen verwendet. (http://msdn.microsoft.com/en-us/library/bb429619.aspx).
  • Die Markierungsnadeln in den Karten werden mit Hilfe des Map Icon Makers von Pamela Fox erzeugt. Die Verwaltung der Markierungen erfolgt mit den MarkerClusterern von Xiaoxi Wu und Luke Mahe.
  • Auf verschiedene Kartendienste wie OpenStreetMap oder esri-Karten wird mit Hilfe des Leaflet-API von Vladimir Agafonkin zugegriffen (http://leafletjs.com)
  • Die MediaUtil von Dmitriy Rogatkin und Suresh Mahalingam wird verwendet, um Miniaturansichten verlustfrei zu rotieren (http://mediachest.sourceforge.net/mediautil).
  • Das jdic-Projekt erlaubt die Integration mit dem E-Mail-Client der Plattform (https://jdic.dev.java.net).
  • iText wird für die Erzeugung von PDF-Dokumenten verwendet. (www.lowagie.com/iText).
  • Lire von SemanticMetadata.net ist die Komponente für die Ähnlichkeitsrecherche (www.semanticmetadata.net/lire).
  • Apache Lucene wird für die Volltextsuche verwendet (http://lucene.apache.org).
  • Der Upload von Fotos zu locr und smugmug basiert auf jUploadr  (http://juploadr.org/)
  • Scribe wird für die Authorisierung von Zugriffen zu Web-Communities benutzt (https://github.com/fernandezpablo85/scribe-java).
  • Der Upload zu flickr erfolgt mit Flickr4Java (https://github.com/callmeal/Flickr4Java).
  • Die Gallerific-Webengine stammt von Trent Foley (www.twospy.com/galleriffic).
  • Die Image Gallery-Webengine stammt von Tony Yoo (http://tonyyoo.com).
  • Die Pirobox-Webengine stammt von Pirolab Webdesign (http://www.pirolab.it).
  • Die Galleria-Webengine stammt von Aino (http://www.aino.se/).
  • Die SimpleViewer- und TiltViewer-Webengines stammen von SimpleViewer Inc. (www.simpleviewer.net).
  • Der Generator für 3D-Virtual-Reality Online-Ausstellungen basiert auf den Arbeiten des The National Museum of China (http://www.visitnmc.com/) und von Gerard Fernandez (www.dhteumeuleu.com) 
  • Die FTP-Unterstützung basiert auf Apache Commons Net (http://commons.apache.org/net/)
  • Hunspell Spell Checker von László Németh (http://hunspell.sourceforge.net/)
  • Das Java-Binding für Hunspell stammt von Flemming Frandsen (http://dren.dk/hunspell.html)
  • Java Native Access (JNA) von Timothy Wall und anderen.
  • SQLite-Zugriff auf fremde Kataloge von David Crawshow https://bitbucket.org/xerial/sqlite-jdbc
  • Diagrammgenerator David Gilbert et al (http://www.jfree.org/)
  • Die Druckerunterstützung stammt vom PaperClips-Projekt von Matthew Hall (http://code.google.com/p/swt-paperclips/)
  • Jama von The MathWorks und dem National Institute of Standards and Technology (NIST) wird für Matrizenalgebra verwendet. (http://math.nist.gov/javanumerics/jama/).
  • Vollbildmodus von Ugo Sangiorgi und anderen (http://ugosan.org/).
  • Der Webserver basiert auf dem HttpServer von Amichai Rothman (http://www.freeutils.net/source/jlhttp/).
  • Dropzone in Webseiten von Matias Meno (http://www.dropzonejs.com)
  • QR code: Project Nayuki (https://www.nayuki.io/page/qr-code-generator-library)
  • Die meisten Icons stammen von Mark James (http://www.famfamfam.com/lab/icons/silk/) und Damien Guard (http://www.damieng.com/icons/silkcompanion)
  • Dank für neue Ideen geht an Johannes Haberl
  • Logo-Entwurf von Alice Philipp
