Capture One Support

ZoRa Photo Director is able to reflect most of the development settings for RAW and DNG files made with Capture One.

These settings (recipes) are stored either in the CaptureOne/ subfolder of the image folder in form of .cos files or - in case of the .eip file format - are embedded in the image file. ZoRa Photo Director is able to interpret these development recipes and to apply them when importing RAW or DNG files, refreshing or viewing them, or when deriving images from them during export operations.

To enable this support go the Window>Preferences>Import>Raw Conversion* and check the Capture One recipe detector. Make also sure that recipe detection is enabled (Process recipes when importing RAW images).

Capture One saves this information incrementally in the moment a change is applied. When the image happens to be located in a watched folder (see Catalog Properties), ZoRa Photo Director will apply these changes either as soon as Capture One applies changes to another image (indicating the end of the editing session for the current image) or when control returns to ZoRa Photo Director.

Unfortunately, Capture One support ceases with Capture One 7. Capture One 7 is not able to open single images, thus cannot be invoked as an image editor form ZoRa Photo Director.
