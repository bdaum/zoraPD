Webserver

The built-in webserver is an optional component. If installed it can expose images, generated exhibitions and generated webgalleries as web contents. The purpose of this webserver is to allow access to the catalog contents from "small" devices such as mobile phones, projectors, or TV sets, basically anything with a webbrowser capable of HTML 5, but not to supply a wide public audience.

Please note that

  • only images with the required privacy level will be shown. Non-JPEG-images and images with a color space other than sRGB are converted to JPEG images with a sRGB color space. Video thumbnails are shown but videos are not displayed..
  • only exhibitions with the required privacy level will be shown.It is also required that the exhibition was successfully generated and exists at the specified output location. If not they will be run as a slideshow.
  • only web galleries with the required privacy level will be shown.It is also required that the web gallery was successfully generated and exists at the specified output location. If not they will be run as a slideshow.

To configure the webserver go to Window>Preferences>Webserver.


General

Start and Stop

  • Autostart. If checked the webserver will be started automatically when the application is started.
  • Start. The button is used to start the webserver manually.
  • Stop. This button is used to stop the webserver manually.

HTTP settings

  • Port. Used to set the port number fior Internet access. Defaults to 8080. E.g. from the same machine the webserver is accessed with http://localhost:8080.
  • Images path. This is the path to the page with the image thumbnails. Defaults to images. Direct access is possible, for instance, with http://localhost:8080/images/.
  • Exhibitions path. This is the path to the page with the exhibition thumbnails. Defaults to exhibitions. Direct access is possible, for instance, with http://localhost:8080/exhibitions/.
  • Web galleries path. This is the path to the page with the web galleries thumbnails. Defaults to galleries. Direct access is possible, for instance, with http://localhost:8080/galleries/.
  • Slideshows path. This is the path to the page with the slideshows thumbnails. Defaults to slideshows. Direct access is possible, for instance, with http://localhost:8080/slideshows/.
Please note that all changes to the HTTP settings require a restart of the webserver.

IP addresses

This section is to inform you about the IP addresses provided by your platform. Select an appropriate IP address and point your smartphone to the QR code to establish a connection.

Contents

Acess rights

  • Chapters. Here you can define which chapters (Images, Exhibitions, Web galleries) are visible on the home page.
  • Groups. Here you can define which groups (Ratings, Directories, Time line, etc.) shall be visible on the overview page.
  • Actions. Here you can define which actions (Show metadata, Show fullsize, Download image file, Show location in map) are made available as buttons below the displayed image.
  • Privacy. Determines the privacy level of images, exhibitions and web galleries required to be displayed.

Contents

  • Thumbnails per page. Determines the maximum number of thumbnails shown on one page. Defaults to 15.
  • Include orphans. If checked, also catalog entries without a high resolution image or without a generated gallery or web exhibition are shown.
  • File types. Only images with the marked file formats will be shown. Defaults to JPEG.
  • Images title This is the title of the page with the image thumbnails. Defaults to Images. 
  • Exhibitions title This is the title of the page with the exhibition thumbnails. Defaults to Exhibitions. 
  • Web galleries title This is the title of the page with the web galleries thumbnails. Defaults to Web galleries.
  • Slideshows title This is the title of the page with the slideshows thumbnails. Defaults to Slideshows.

Output quality

  • Sharpen. Check this button to enable output sharpening. An Unsharp Mask filter is used for that purpose:
    • Amount. The amount of sharpening
    • Radius. The radius of the Unsharp Mask in pixel.
    • Threshhold. Increase this value for inhibiting sharpening in low contrast area, so that noise is kept low.
Downsizing of images for the details view or slideshows causes smear between pixels. Sharpening helps to improve the look of the image.

Uploads

It is optionally possible to upload images and videos for the web browser to the catalog. This feature can be enabled by checking the option Allow uploads-

Access control

Optionally, you may specify a password. If specified users will need to enter that password before getting access to the upload facility.
You may also restict the user by unchekcing Allow metadata modification. If this option is enabled the user may specify metadata for the uploaded files such as artist, event, keywords, and timeshift. In addition she may even upload XMP sidecars with the uploaded files.

Import channel

If uploads are enabled images are first uploaded to the specified Transfer folder. ZoRa PhotoDirector will import the images in the transfer folder according to the specifications given by the transfer folder. If no transfer folder exists or you need a different transfer folder, click the Add... button.

HTML

This tab allows you editing the webservers HTML pages. Currently, there are three different pages: the home page, a overview page with the thumbnails and a navigation section, and the details page displaying a single image and the metadata.
However, you should leave the structure of these files intact. Editing is only recommended for the natural language text embedded in those files.
Undo and Redo is performed - as usual - using Ctrl+Z and Ctrl+Y.
By pressing the Reset button you can reset the current file to its original content.

CSS

The CSS stylesheet applies to all HTML pages. It can be edited to change the Look&Feel auf those pages.
Undo and Redo is performed - as usual - using Ctrl+Z and Ctrl+Y.
By pressing the Reset button you can reset the CSS stylesheet to its original content.
